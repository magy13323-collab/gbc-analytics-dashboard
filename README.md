# Тестовое задание AI Tools Specialist

## Что сделано

1. Создан демо аккаунт в RetailCRM.
2. Создан проект в Supabase.
3. Загружены 50 тестовых заказов в RetailCRM.
4. Написан скрипт синхронизации RetailCRM в Supabase.
5. Создан дашборд с графиком заказов на основе Supabase.
6. Проект задеплоен в Vercel.
7. Telegram уведомления настроены и протестированы на заказе больше 50000.
8. Для проверки Telegram уведомлений и повторной синхронизации были созданы дополнительные тестовые заказы, поэтому итоговое количество заказов в системе стало 52.

## Структура проекта

1. `import_one_order.py`  
   Тестовая загрузка одного заказа в RetailCRM.

2. `import_all_orders.py`  
   Массовая загрузка заказов из `mock_orders.json` в RetailCRM.

3. `import_remaining_orders.py`  
   Продолжение импорта заказов после сбоя.

4. `sync_retailcrm_to_supabase.py`  
   Синхронизация заказов из RetailCRM в таблицу `orders` в Supabase.

5. `import_big_order.py`  
   Создание тестового большого заказа для проверки Telegram уведомления.

6. `telegram_notifier.py`  
   Проверка новых заказов в RetailCRM и отправка уведомлений в Telegram.

7. `index.html`  
   Минимальная веб страница дашборда с графиком.

8. `api`  
   Серверный API роут для получения агрегированных данных из Supabase.

9. `lib`  
   Вспомогательная логика агрегации данных для API.

## Дашборд

Сейчас график показывает количество заказов по минутам создания заказа, потому что тестовые заказы были импортированы за короткий промежуток времени.

## Как запустить локально

1. Создать файл `.env` в корне проекта.
2. Указать переменные `RETAILCRM_URL`, `RETAILCRM_API_KEY`, `RETAILCRM_SITE`, `SUPABASE_URL`, `SUPABASE_KEY`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`.
3. Запустить команду `npm run dev`.
4. Открыть `http://localhost:3000`.

## Где возникли проблемы и как решал

1. Ошибка с полями `orderType`, `orderMethod`, `status`.  
   Удалил эти поля перед отправкой в RetailCRM.

2. Таймаут при массовом импорте.  
   Добавил запуск продолжения импорта с нужного индекса и паузы между запросами.

3. Ошибка соединения при записи в Supabase.  
   Добавил батчевую загрузку и обработку ошибок по партиям.

4. Конфликт по первичному ключу при повторной загрузке.  
   Использовал upsert с `on_conflict=retailcrm_id`.

5. Неинформативный график при агрегации по дням.  
   Поменял агрегацию на минуты.

6. Для проверки Telegram уведомлений создал отдельный тестовый заказ на сумму 70000.
7. После финальной проверки синхронизации исправил заголовок Prefer для корректного upsert в Supabase и временно уменьшил batch_size до 1, чтобы избежать обрыва соединения при загрузке.
