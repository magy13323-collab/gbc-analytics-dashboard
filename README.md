# Тестовое задание AI Tools Specialist

## Что сделано
- Создан демо-аккаунт в RetailCRM.
- Создан проект в Supabase.
- Загружены 50 тестовых заказов в RetailCRM.
- Написан скрипт синхронизации RetailCRM в Supabase.
- Создан дашборд с графиком заказов на основе Supabase.
- Проект подготовлен к деплою в Vercel.
- Telegram-уведомления будут настроены отдельным шагом.

## Структура проекта
- `import_one_order.py` — тестовая загрузка одного заказа в RetailCRM.
- `import_all_orders.py` — массовая загрузка заказов из `mock_orders.json` в RetailCRM.
- `import_remaining_orders.py` — продолжение импорта заказов после сбоя.
- `sync_retailcrm_to_supabase.py` — синхронизация заказов из RetailCRM в таблицу `orders` в Supabase.
- `index.html` — минимальная веб-страница дашборда с графиком.
- `api` — серверный API-роут для получения агрегированных данных из Supabase.
- `lib` — вспомогательная логика агрегации данных для API.

## Дашборд
Сейчас график показывает количество заказов по минутам создания заказа, потому что тестовые заказы были импортированы за короткий промежуток времени.

## Как запустить локально
1. Создать файл `.env` в корне проекта.
2. Указать переменные:
   `RETAILCRM_URL`, `RETAILCRM_API_KEY`, `RETAILCRM_SITE`, `SUPABASE_URL`, `SUPABASE_KEY`.
3. Запустить:
   `npm run dev`
4. Открыть:
   `http://localhost:3000`

## Где возникли проблемы и как решал
- Ошибка с полями `orderType`, `orderMethod`, `status` — удалил эти поля перед отправкой в RetailCRM.
- Таймаут при массовом импорте — добавил запуск продолжения импорта с нужного индекса и паузы между запросами.
- Ошибка соединения при записи в Supabase — добавил батчевую загрузку и обработку ошибок по партиям.
- Конфликт по первичному ключу при повторной загрузке — использовал upsert с `on_conflict=retailcrm_id`.
- Неинформативный график при агрегации по дням — поменял агрегацию на минуты.
